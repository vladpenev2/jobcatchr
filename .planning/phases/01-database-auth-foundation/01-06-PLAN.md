<task id="01-06-01">
<title>Create admin guard layout (src/app/(app)/admin/layout.tsx)</title>
<details>
Create `src/app/(app)/admin/layout.tsx` that protects all /admin routes.

This is a Server Component that:
1. Creates a Supabase server client
2. Calls `getUser()` to get the authenticated user
3. Queries the profiles table for the user's role
4. If not admin, redirects to `/`
5. If admin, renders children

CRITICAL: Use `getUser()` not `getClaims()`.

The admin check is done at the layout level (not in the proxy) to avoid a DB query on every request. The proxy only handles authentication (is the user logged in?). The admin layout handles authorization (is the user an admin?).

```typescript
import { createClient } from '@/lib/supabase/server'
import { redirect } from 'next/navigation'

export default async function AdminLayout({ children }: { children: React.ReactNode }) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()

  if (!user) redirect('/login')

  const { data: profile } = await supabase
    .from('profiles')
    .select('role')
    .eq('id', user.id)
    .single()

  if (profile?.role !== 'admin') redirect('/')

  return <>{children}</>
}
```

Note: The parent (app) layout already handles the nav and checks auth. This layout only adds the admin role check. It renders children directly without additional wrapping.

<task id="01-06-03">
<title>Create unit test for admin guard logic</title>
<details>
Create `src/__tests__/admin-guard.test.ts` that tests the admin authorization logic.

Since async Server Components can't be directly rendered in Vitest, extract the admin check logic into a testable helper function, OR test the logic pattern in isolation.

Approach: Test the admin check logic by mocking the Supabase client and verifying the redirect behavior.

```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest'

// Mock env
vi.stubEnv('NEXT_PUBLIC_SUPABASE_URL', 'https://test.supabase.co')
vi.stubEnv('NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY', 'sb_publishable_test')

const mockGetUser = vi.fn()
const mockFrom = vi.fn()
const mockRedirect = vi.fn()

vi.mock('@/lib/supabase/server', () => ({
  createClient: vi.fn(async () => ({
    auth: { getUser: mockGetUser },
    from: mockFrom,
  })),
}))

vi.mock('next/navigation', () => ({
  redirect: (...args: any[]) => {
    mockRedirect(...args)
    throw new Error('NEXT_REDIRECT') // redirect throws in Next.js
  },
}))

vi.mock('next/headers', () => ({
  cookies: vi.fn(async () => ({
    getAll: vi.fn(() => []),
    set: vi.fn(),
  })),
}))

describe('Admin guard logic', () => {
  beforeEach(() => {
    vi.clearAllMocks()
  })

  it('redirects to /login when no user', async () => {
    mockGetUser.mockResolvedValue({ data: { user: null } })

    // Import and call the layout function
    const { default: AdminLayout } = await import('@/app/(app)/admin/layout')

    try {
      await AdminLayout({ children: null })
    } catch (e: any) {
      if (e.message !== 'NEXT_REDIRECT') throw e
    }

    expect(mockRedirect).toHaveBeenCalledWith('/login')
  })

  it('redirects to / when user is not admin', async () => {
    mockGetUser.mockResolvedValue({ data: { user: { id: 'user-1' } } })
    mockFrom.mockReturnValue({
      select: vi.fn().mockReturnValue({
        eq: vi.fn().mockReturnValue({
          single: vi.fn().mockResolvedValue({ data: { role: 'user' } }),
        }),
      }),
    })

    const { default: AdminLayout } = await import('@/app/(app)/admin/layout')

    try {
      await AdminLayout({ children: null })
    } catch (e: any) {
      if (e.message !== 'NEXT_REDIRECT') throw e
    }

    expect(mockRedirect).toHaveBeenCalledWith('/')
  })

  it('renders children when user is admin', async () => {
    mockGetUser.mockResolvedValue({ data: { user: { id: 'admin-1' } } })
    mockFrom.mockReturnValue({
      select: vi.fn().mockReturnValue({
        eq: vi.fn().mockReturnValue({
          single: vi.fn().mockResolvedValue({ data: { role: 'admin' } }),
        }),
      }),
    })

    const { default: AdminLayout } = await import('@/app/(app)/admin/layout')
    const result = await AdminLayout({ children: 'test content' as any })

    expect(mockRedirect).not.toHaveBeenCalled()
    // result should be the rendered children
  })
})
```

Note: Adjust test approach if async Server Components don't work well with Vitest. The key is to verify that the role check logic correctly redirects non-admin users and passes admin users through. If direct testing of the layout component is flaky, extract the check into a `checkAdminAccess()` utility in `src/lib/auth.ts` and test that instead.
