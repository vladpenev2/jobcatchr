<task id="01-05-03">
<title>Create login page</title>
<details>
Create `src/app/login/page.tsx` - a centered login form with email and password fields.

Design:
- Dark theme (inherits from root)
- Centered Card component on the page
- "Job Catchr" heading or logo text at top
- Email input field
- Password input field
- "Sign In" button (full width)
- Error message display area (for invalid credentials)
- No registration link (registrations are closed)

Use a Client Component with useState for form state and error handling. Call a Server Action for the actual login.

The login form should:
1. Accept email and password
2. Call Supabase auth via a Server Action (src/app/login/actions.ts)
3. On success: redirect to `/` (the proxy handles the rest)
4. On error: display error message

Create `src/app/login/actions.ts` as a Server Action:
```typescript
'use server'

import { createClient } from '@/lib/supabase/server'
import { redirect } from 'next/navigation'

export async function login(formData: FormData) {
  const supabase = await createClient()
  const { error } = await supabase.auth.signInWithPassword({
    email: formData.get('email') as string,
    password: formData.get('password') as string,
  })
  if (error) {
    return { error: error.message }
  }
  redirect('/')
}
```

The login page uses a native HTML `<form>` with the Server Action bound via `useActionState` (React 19). This is intentional - do NOT use react-hook-form and do NOT install the shadcn `form` component. The `useActionState` hook manages form state (pending, error) without any form library. The shadcn components installed in task 01-05-01 are sufficient.
</details>

<task id="01-05-06">
<title>Create (app) route group layout with nav shell</title>
<details>
Create the `src/app/(app)/layout.tsx` route group layout that wraps all authenticated routes.

This is a Server Component that:
1. Creates a Supabase server client
2. Calls `getUser()` to get the authenticated user (the proxy already ensured they're logged in, but we need the user data for the nav)
3. Queries the `profiles` table to get the user's name and role
4. Renders the TopNav with user info
5. Renders children in a main content area

```typescript
import { createClient } from '@/lib/supabase/server'
import { redirect } from 'next/navigation'
import { TopNav } from '@/components/nav/top-nav'

export default async function AppLayout({ children }: { children: React.ReactNode }) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()

  if (!user) redirect('/login')

  const { data: profile } = await supabase
    .from('profiles')
    .select('name, email, role')
    .eq('id', user.id)
    .single()

  return (
    <div className="min-h-screen bg-background">
      <TopNav
        userName={profile?.name ?? user.email ?? 'User'}
        userEmail={profile?.email ?? user.email ?? ''}
        isAdmin={profile?.role === 'admin'}
      />
      <main className="container mx-auto py-6 px-4">
        {children}
      </main>
    </div>
  )
}
```

CRITICAL: Use `getUser()` not `getClaims()` for the auth check.
</details>

<task id="01-05-07">
<title>Create app home page placeholder and delete root page</title>
<details>
Create `src/app/(app)/page.tsx` as the authenticated home page. For now, a simple placeholder:

```tsx
export default function HomePage() {
  return (
    <div>
      <h1 className="text-2xl font-bold">Jobs</h1>
      <p className="text-muted-foreground mt-2">
        Job search and listing will be built in Phase 3.
      </p>
    </div>
  )
}
```

DELETE `src/app/page.tsx` entirely. Do not replace it with a redirect or any other content - just delete the file.

Why: `(app)/page.tsx` maps to `/` directly (route groups are transparent and add no path segments). Having both `src/app/page.tsx` and `src/app/(app)/page.tsx` causes a Next.js conflict at the `/` route. The `(app)/layout.tsx` handles the auth check and redirects unauthenticated users to `/login`. The middleware proxy also redirects unauthenticated requests to `/login` before they reach any page component. Deleting `src/app/page.tsx` resolves the conflict cleanly.
</details>
