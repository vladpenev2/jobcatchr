# Plan 02: Database Schema Migration

<objective>
Fix the pg_trgm ordering bug in schema.sql and run the full migration against Supabase to create all 9 tables, enums, indexes, RLS policies, functions, and triggers.
</objective>

<tasks>

<task id="01-02-01">
<title>Fix pg_trgm extension ordering in schema.sql</title>
<details>
The schema has a bug: trigram indexes (lines 175-176) are created BEFORE the `pg_trgm` extension is enabled (line 198). This will fail at runtime.

Move `create extension if not exists pg_trgm;` to the very top of the file, before the ENUMS section. Add it right after the header comment.

The fixed ordering should be:
1. `create extension if not exists pg_trgm;` (TOP)
2. ENUMS
3. TABLES
4. INDEXES (including trigram indexes - now safe)
5. FUNCTIONS
6. RLS
7. SEED comments

Remove the original `create extension` line from the TRIGRAM EXTENSION section and delete that section header since it is no longer needed as a separate section.
</details>

<task id="01-02-02">
<title>Run schema migration via Supabase</title>
<details>
Execute the corrected schema.sql against the Supabase database. Use the Supabase MCP `supabase_execute_sql` tool to run the entire SQL file as a single statement.

If the MCP tool is not available, use the Supabase Dashboard SQL Editor to paste and run the full schema.

The schema creates:
- 5 enums: user_role, job_status, job_source, time_range, schedule_interval
- 9 tables: profiles, jobs, searches, search_jobs, user_saved_jobs, user_job_views, glassdoor_cache, company_cache, people_searches
- 12+ indexes including trigram indexes
- pg_trgm extension
- update_updated_at() trigger function + triggers on profiles and jobs
- is_admin() security definer function
- RLS enabled on all 9 tables with policies
</details>

<task id="01-02-03">
<title>Verify RLS and extensions</title>
<details>
After migration, verify:

1. RLS is enabled on all tables:
```sql
SELECT tablename, rowsecurity FROM pg_tables WHERE schemaname = 'public' ORDER BY tablename;
```
All 9 tables should have `rowsecurity = true`.

2. pg_trgm extension is active:
```sql
SELECT * FROM pg_extension WHERE extname = 'pg_trgm';
```

3. is_admin() function exists:
```sql
SELECT routine_name FROM information_schema.routines WHERE routine_schema = 'public' AND routine_name = 'is_admin';
```

4. Triggers exist:
```sql
SELECT trigger_name, event_object_table FROM information_schema.triggers WHERE trigger_schema = 'public';
```
Should show profiles_updated_at and jobs_updated_at.
</details>
