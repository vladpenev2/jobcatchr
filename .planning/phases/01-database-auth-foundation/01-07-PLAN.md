<details>

<task id="01-07-03">
<title>Admin API routes</title>
<details>
Create three API routes using the service role Supabase client:

**GET/POST `src/app/api/admin/users/route.ts`**:
- GET: List all users from `profiles` table (name, email, location, role, profile_synced_at, created_at)
- POST: Create a new user
  1. Generate random 12-char password
  2. Create auth user via `supabase.auth.admin.createUser({ email, password, email_confirm: true })`
  3. Insert into `profiles` table (name, email, linkedin_url, location, role: 'user')
  4. Call `extractLinkedInProfile(linkedin_url)` and update `profiles.profile_data` + `profile_synced_at`
  5. Return the generated password (shown once to admin)

**PUT `src/app/api/admin/users/[id]/route.ts`**:
- Update user name, location

**POST `src/app/api/admin/users/[id]/sync-profile/route.ts`**:
- Re-extract LinkedIn profile and update profile_data + profile_synced_at

All routes must check `is_admin` by querying the caller's profile role. Return 403 if not admin.
</details>
<files>
- src/app/api/admin/users/route.ts (create)
- src/app/api/admin/users/[id]/route.ts (create)
- src/app/api/admin/users/[id]/sync-profile/route.ts (create)
</files>
