# Plan 01: Vitest Setup & Test Infrastructure

<objective>
Install Vitest and testing dependencies, create the config file with React plugin and path aliases, and add a smoke test that verifies the setup works end-to-end.
</objective>

<tasks>

<task id="01-01-01">
<title>Install Vitest and testing dependencies</title>
<details>
Run:
```bash
npm install -D vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/dom vite-tsconfig-paths
```

Then add test scripts to package.json:
```json
"test": "vitest",
"test:run": "vitest run"
```
</details>

- package.json (modify)
</files>
<verify>
<automated>npx vitest --version</automated>
</verify>
</task>

<task id="01-01-02">
<title>Create vitest.config.mts</title>
<details>
Create `vitest.config.mts` at the project root with:
- `@vitejs/plugin-react` for JSX/TSX support
- `vite-tsconfig-paths` to resolve the `@/*` path alias from tsconfig.json
- `jsdom` test environment for DOM testing
- `globals: true` so `describe`, `it`, `expect` are available without imports

```typescript
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'
import tsconfigPaths from 'vite-tsconfig-paths'

export default defineConfig({
  plugins: [tsconfigPaths(), react()],
  test: {
    environment: 'jsdom',
    globals: true,
  },
})
```
</details>

- vitest.config.mts (create)
</files>
<verify>
<manual>File exists with correct plugins and config</manual>
</verify>
</task>

<task id="01-01-03">
<title>Create smoke test for cn utility</title>
<details>
Create `src/__tests__/utils.test.ts` that tests the existing `cn` utility from `@/lib/utils`. This validates the full chain: Vitest runs, path aliases resolve, and the test environment works.

```typescript
import { describe, it, expect } from 'vitest'
import { cn } from '@/lib/utils'

describe('cn utility', () => {
  it('merges class names', () => {
    expect(cn('foo', 'bar')).toBe('foo bar')
  })

  it('handles conditional classes', () => {
    expect(cn('base', false && 'hidden', 'visible')).toBe('base visible')
  })

  it('merges tailwind conflicts correctly', () => {
    expect(cn('px-4', 'px-6')).toBe('px-6')
  })
})
```
</details>
