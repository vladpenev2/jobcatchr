</objective>

<tasks>

<task id="01-04-01">
<title>Create updateSession helper (src/lib/supabase/proxy.ts)</title>
<details>
Create `src/lib/supabase/proxy.ts` with the `updateSession` function. This is the core auth logic that:

1. Creates a Supabase server client with request/response cookie bridging
2. Calls `supabase.auth.getUser()` to validate the JWT (NEVER use getSession() or getClaims() server-side)
3. Redirects unauthenticated users to `/login` (except for /login and /auth routes)
4. Redirects authenticated users away from `/login` to `/`
5. Returns the response with refreshed cookies

CRITICAL: Use `getUser()` not `getClaims()` for auth validation. `getUser()` makes a network call to Supabase Auth to validate the JWT. `getClaims()` and `getSession()` only read from local storage without validation.

```typescript
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function updateSession(request: NextRequest) {
  let supabaseResponse = NextResponse.next({ request })

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY!,
    {
      cookies: {
        getAll() {
          return request.cookies.getAll()
        },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value))
          supabaseResponse = NextResponse.next({ request })
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, options)
          )
        },
      },
    }
  )

  const { data: { user } } = await supabase.auth.getUser()

  // Unauthenticated -> redirect to /login
  if (
    !user &&
    !request.nextUrl.pathname.startsWith('/login') &&
    !request.nextUrl.pathname.startsWith('/auth')
  ) {
    const url = request.nextUrl.clone()
    url.pathname = '/login'
    return NextResponse.redirect(url)
  }

  // Authenticated on /login -> redirect to /
  if (user && request.nextUrl.pathname.startsWith('/login')) {
    const url = request.nextUrl.clone()
    url.pathname = '/'
    return NextResponse.redirect(url)
  }

  return supabaseResponse
}
```

Note: Admin route protection is NOT in the proxy. It is handled at the layout level (Plan 06) to keep the proxy fast and avoid DB queries on every request.

<task id="01-04-02">
<title>Create proxy.ts entry point (src/proxy.ts)</title>
<details>
Create `src/proxy.ts` - the Next.js 16 proxy file (replaces middleware.ts).

CRITICAL: Next.js 16 renamed middleware to proxy. The file is `src/proxy.ts` and exports `function proxy()` not `function middleware()`.

```typescript
import { type NextRequest } from 'next/server'
import { updateSession } from '@/lib/supabase/proxy'

export async function proxy(request: NextRequest) {
  return await updateSession(request)
}

export const config = {
  matcher: [
    /*
     * Match all paths except:
     * - _next/static (static files)
     * - _next/image (image optimization)
     * - favicon.ico
     * - Static assets (svg, png, jpg, etc.)
     */
    '/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)',
  ],
}
```

The matcher excludes static assets from the proxy to avoid unnecessary auth checks on every image/CSS/JS file.

<task id="01-04-03">
<title>Create unit test for updateSession logic</title>
<details>
Create `src/__tests__/proxy.test.ts` to test the redirect logic of `updateSession`.

Mock `@supabase/ssr` createServerClient to control what `getUser()` returns. Mock `next/server` NextResponse.

Test cases:
1. Unauthenticated user visiting `/` -> redirects to `/login`
2. Unauthenticated user visiting `/login` -> passes through (no redirect)
3. Authenticated user visiting `/login` -> redirects to `/`
4. Authenticated user visiting `/` -> passes through (no redirect)
5. Unauthenticated user visiting `/auth/callback` -> passes through

```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest'

// Mock env
vi.stubEnv('NEXT_PUBLIC_SUPABASE_URL', 'https://test.supabase.co')
vi.stubEnv('NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY', 'sb_publishable_test')

const mockGetUser = vi.fn()

vi.mock('@supabase/ssr', () => ({
  createServerClient: vi.fn(() => ({
    auth: { getUser: mockGetUser },
  })),
}))

// Build minimal NextRequest/NextResponse mocks
function makeRequest(pathname: string) {
  const url = new URL(`http://localhost:3000${pathname}`)
  return {
    nextUrl: { pathname, clone: () => ({ ...url, pathname }) },
    cookies: {
      getAll: vi.fn(() => []),
      set: vi.fn(),
    },
  }
}

// Mock NextResponse
const mockRedirect = vi.fn((url) => ({ type: 'redirect', url }))
const mockNext = vi.fn((...args) => ({
  type: 'next',
  cookies: { set: vi.fn() },
}))

vi.mock('next/server', () => ({
  NextResponse: {
    redirect: (...args: any[]) => mockRedirect(...args),
    next: (...args: any[]) => mockNext(...args),
  },
}))

describe('updateSession', () => {
  beforeEach(() => {
    vi.clearAllMocks()
  })

  it('redirects unauthenticated user from / to /login', async () => {
    mockGetUser.mockResolvedValue({ data: { user: null } })
    const { updateSession } = await import('@/lib/supabase/proxy')
    const req = makeRequest('/')
    await updateSession(req as any)
    expect(mockRedirect).toHaveBeenCalled()
  })

  it('allows unauthenticated user on /login', async () => {
    mockGetUser.mockResolvedValue({ data: { user: null } })
    const { updateSession } = await import('@/lib/supabase/proxy')
    const req = makeRequest('/login')
    const res = await updateSession(req as any)
    expect(mockRedirect).not.toHaveBeenCalled()
  })

  it('redirects authenticated user from /login to /', async () => {
    mockGetUser.mockResolvedValue({ data: { user: { id: 'user-1' } } })
    const { updateSession } = await import('@/lib/supabase/proxy')
    const req = makeRequest('/login')
    await updateSession(req as any)
    expect(mockRedirect).toHaveBeenCalled()
  })

  it('allows authenticated user on /', async () => {
    mockGetUser.mockResolvedValue({ data: { user: { id: 'user-1' } } })
    const { updateSession } = await import('@/lib/supabase/proxy')
    const req = makeRequest('/')
    const res = await updateSession(req as any)
    expect(mockRedirect).not.toHaveBeenCalled()
  })
})
```

Note: Since module mocks are cached per test file, use `vi.resetModules()` in beforeEach if needed, or use dynamic imports with `vi.importActual` patterns. Adjust mocking strategy as needed to get reliable results.
