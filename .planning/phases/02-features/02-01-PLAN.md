<task id="02-01-04">
<title>Search controller with dedup and upsert</title>
<details>
Create `src/lib/adapters/search-controller.ts`:

`executeSearch(criteria: SearchCriteria, userId: string, onProgress: (event: SearchProgress) => void): Promise<{ totalNew: number, totalUpdated: number, searchId: string }>`

Flow:
1. Create a search record in `searches` table
2. Fan out to enabled adapters in parallel (Promise.allSettled)
3. Stream progress events via onProgress callback
4. Merge all results
5. Deduplicate by job `id`
6. Upsert to `jobs` table: INSERT ... ON CONFLICT (id) DO UPDATE SET ... (update all fields except id, created_at)
7. Link jobs to search via `search_jobs` junction table
8. Update search record with job_count and last_run_at
9. Return counts

Use the admin Supabase client (service role) for writes since jobs table RLS only allows admin writes.
</details>

<task id="02-01-05">
<title>SSE search endpoint</title>
<details>
Create `src/app/api/search/route.ts` as a POST endpoint that returns an SSE stream.

Pattern:
```typescript
export async function POST(request: Request) {
  const criteria = await request.json()
  const user = // get from supabase auth

  const encoder = new TextEncoder()
  const stream = new ReadableStream({
    async start(controller) {
      const send = (event: SearchProgress) => {
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(event)}\n\n`))
      }

      try {
        const result = await executeSearch(criteria, user.id, send)
        send({ stage: 'complete', status: 'done', ...result })
      } catch (error) {
        send({ stage: 'error', status: 'error', message: error.message })
      } finally {
        controller.close()
      }
    }
  })

  return new Response(stream, {
    headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', 'Connection': 'keep-alive' }
  })
}
```
</details>
