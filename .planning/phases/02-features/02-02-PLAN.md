# Plan 02: Job List & Detail + Glassdoor

<objective>
Build the main job browsing experience: data table with tabs (All/Saved), filtering, sorting, pagination, job detail side drawer with About/Company/Glassdoor tabs, and all job-related API routes including Glassdoor review fetching.
</objective>

<tasks>

<task id="02-02-01">
<title>Job API routes</title>
<details>
Install shadcn: `npx shadcn@latest add sheet tabs scroll-area select checkbox`

Create API routes:

**GET `src/app/api/jobs/route.ts`**:
- Query params: tab (all|saved), search (text), status (active|likely_expired|expired), source (career-site|linkedin), sort (posted|title|company), order (asc|desc), page, limit
- For "all" tab: get all jobs linked to user's searches via search_jobs junction
- For "saved" tab: get jobs from user_saved_jobs
- Join with user_job_views to get "seen" status
- Join with user_saved_jobs to get "saved" status
- Return paginated results with total count
- Use server Supabase client (RLS handles user filtering)

**GET `src/app/api/jobs/[id]/route.ts`**:
- Return full job detail

**DELETE `src/app/api/jobs/[id]/route.ts`**:
- Delete job (admin only or soft delete)

**POST `src/app/api/jobs/[id]/view/route.ts`**:
- Upsert user_job_views: insert or update last_viewed_at

**POST `src/app/api/jobs/[id]/save/route.ts`**:
- Toggle: if saved, delete from user_saved_jobs. If not, insert.
- Return new save state.
</details>

<task id="02-02-04">
<title>Glassdoor reviews integration</title>
<details>
Create `src/lib/apify/glassdoor.ts`:
- `fetchGlassdoorReviews(companyName: string): Promise<{ reviews, rating_overall, review_count, employer_logo_url }>`
- Uses Apify actor: `getdataforme/glassdoor-reviews-scraper`
- Input: `{ company_name: companyName, item_limit: 15 }`
- Reference scope/glassdoor_reviews_scraper.md for output schema

Create `src/app/api/jobs/[id]/glassdoor/route.ts`:
- GET: check glassdoor_cache for company name (lowercased)
- If cached and < 30 days old: return from cache
- If not: call fetchGlassdoorReviews, store in cache, return
- Use admin client for cache writes

Create `src/components/jobs/glassdoor-tab.tsx` (Client Component):
- Fetches reviews on tab click (lazy load)
- Shows overall rating (X/5 stars, N reviews)
- List of review cards: title, rating stars, job title + location, employment status + tenure, pros (green), cons (red), advice, date
- Loading state while fetching
- "No reviews found" empty state
</details>
